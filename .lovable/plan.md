

# خطة شاملة: تقرير PDF للعروض + إشعارات تلقائية + تحسين Dashboard

## 1. التحقق من لوحة الإحصائيات

لوحة الإحصائيات في `QuotationUpload.tsx` تعمل بشكل صحيح بالفعل:
- `quotationStats` (سطر 162-169) يحسب: `suppliers` (عدد الموردين) و `avgItemPrice` (متوسط سعر البند)
- الشبكة (سطر 1406-1449) تعرض 6 بطاقات تشمل الموردين ومتوسط السعر
- **لا يحتاج تعديل** - كل شيء يعمل

## 2. تصدير تقرير PDF شامل لجميع عروض الأسعار المحللة

### الملف: `src/components/QuotationUpload.tsx`

اضافة دالة `exportComprehensiveQuotationsPDF` تنشئ تقرير PDF متعدد الصفحات يتضمن:

**صفحة الغلاف:**
- عنوان "تقرير عروض الأسعار الشامل"
- تاريخ التقرير وعدد العروض

**صفحة الإحصائيات:**
- جدول ملخص (عدد العروض، المحللة، البنود، القيمة، الموردين، المتوسط)
- جدول مقارنة الموردين (اسم المورد، عدد البنود، إجمالي القيمة)

**صفحات تفصيلية لكل عرض:**
- معلومات المورد
- جدول البنود (رقم، وصف، وحدة، كمية، سعر، إجمالي)
- إجمالي القيمة

**صفحة مقارنة الأسعار:**
- جدول يقارن أسعار البنود المتشابهة بين العروض المختلفة

### UI:
- إضافة زر "تصدير تقرير PDF شامل" بجانب زر "استيراد الكل إلى المكتبة"

## 3. إشعارات تلقائية عند اكتمال التحليل ومقارنة الأسعار

### الملف: `src/components/QuotationUpload.tsx`

**أ) إشعار اكتمال التحليل:**
- بعد نجاح `analyzeQuotation` (سطر 745-749)، إضافة إشعار عبر `useNotifications` hook
- إشعار يتضمن: اسم العرض، عدد البنود، إجمالي القيمة
- إشعار صوتي ومرئي (toast + browser notification عبر `usePushNotifications`)

**ب) مقارنة تلقائية مع المكتبة:**
- بعد التحليل الناجح، مقارنة أسعار البنود مع `material_prices` في المكتبة
- إذا وجد فرق اكبر من 20%، إظهار إشعار تحذيري
- عرض ملخص: "X بنود بأسعار أعلى من المكتبة بنسبة Y%"

### التعديلات:
- استيراد `useNotifications` و `usePushNotifications`
- إضافة دالة `compareWithLibrary(items)` بعد التحليل
- إضافة إشعارات في نهاية `analyzeQuotation` و `handleBatchAnalyze`
