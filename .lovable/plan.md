
# خطة: إصلاح شاشات الإدارة + إضافة نصائح واقتراحات

## المشكلة
عند تسجيل حساب جديد والدخول لصفحات "إدارة الصلاحيات" أو "لوحة الإدارة"، يظهر:
- رسالة "غير مصرح" مع زر "تعيين كمشرف أول" الذي لا يعمل لأن مشرف موجود بالفعل
- لا توجد رسالة واضحة تشرح للمستخدم ماذا يفعل
- لا توجد نصائح استخدام (Tips) لهذه الصفحات

## الحلول المقترحة

### 1. تحسين شاشة "غير مصرح" في الصفحتين
**الملفات:** `AdminDashboardPage.tsx` و `UserManagementPage.tsx`

- فحص وجود مشرف عبر استدعاء `setup-admin` مسبقاً وعرض رسالة مناسبة:
  - إذا لا يوجد مشرف: عرض زر "تعيين كمشرف أول"
  - إذا يوجد مشرف: عرض رسالة "تواصل مع المشرف لمنحك الصلاحية" مع عرض بريده
- إضافة أيقونات وتصميم أوضح للشاشة
- إضافة خطوات إرشادية (Steps) للمستخدم الجديد

### 2. إضافة نصائح الاستخدام (Page Tips)
**الملف:** `src/lib/page-tips.ts`

إضافة tips لمسارين:
- `/admin/dashboard` - نصائح لوحة الإدارة (مراقبة النظام، التقارير الأسبوعية، النشاط اليومي)
- `/admin/users` - نصائح إدارة الصلاحيات (تغيير الأدوار، البحث، سجل النشاطات)

### 3. إضافة `PageTipsBox` في الصفحتين
**الملفات:** `AdminDashboardPage.tsx` و `UserManagementPage.tsx`

- إدراج مكون `PageTipsBox` في أعلى كل صفحة بعد العنوان

### 4. تحسين Edge Function: `setup-admin`
**الملف:** `supabase/functions/setup-admin/index.ts`

- عند وجود مشرف بالفعل، إرجاع بريد المشرف الحالي ليتمكن المستخدم الجديد من التواصل معه
- مثال الاستجابة: `{ error: "admin_exists", admin_email: "m***@example.com" }` (مع إخفاء جزء من البريد)

---

## التفاصيل التقنية

### تعديلات `setup-admin/index.ts`:
```text
عند وجود مشرف:
1. جلب user_id من user_roles
2. جلب البريد من auth.admin.getUserById
3. إخفاء جزء من البريد (masking): "mo***@yahoo.com"
4. إرجاع { error: "admin_exists", admin_email: "mo***@yahoo.com" }
```

### تعديلات شاشة Unauthorized:
```text
- عرض خطوات مرقمة:
  1. هذه الصفحة للمشرفين فقط
  2. تواصل مع المشرف: [بريد مخفي جزئياً]
  3. اطلب منه منحك صلاحية من صفحة إدارة الصلاحيات
- إخفاء زر "تعيين كمشرف أول" عند وجود مشرف
- عرض زر "العودة للرئيسية" فقط
```

### نصائح page-tips.ts:
```text
/admin/dashboard:
- "راقب إحصائيات النظام: المستخدمين، المشاريع، العقود"
- "أرسل تقرير أسبوعي تلقائي من زر التقارير"
- "تابع النشاط اليومي عبر الرسوم البيانية"
- "اضغط على مشروع في الجدول لعرض تفاصيله"

/admin/users:
- "غيّر دور أي مستخدم من القائمة المنسدلة بجانب اسمه"
- "استخدم البحث لإيجاد مستخدم بالبريد الإلكتروني"
- "فلتر المستخدمين حسب الدور: مشرف، مشرف مساعد، مستخدم"
- "راجع سجل النشاطات من صفحة سجل النشاط"
```

### ملخص الملفات المتأثرة:
1. `supabase/functions/setup-admin/index.ts` - إرجاع بريد المشرف
2. `src/pages/AdminDashboardPage.tsx` - تحسين شاشة عدم الصلاحية + Tips
3. `src/pages/UserManagementPage.tsx` - تحسين شاشة عدم الصلاحية + Tips
4. `src/lib/page-tips.ts` - إضافة نصائح المسارين الجديدين
