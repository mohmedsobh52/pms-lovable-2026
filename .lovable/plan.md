
# خطة: إضافة زر استيراد قالب جاهز من ملفات متعددة

## الوصف
إضافة زر "استيراد قالب" في قسم "قوالب التحليل" يدعم استيراد ملفات Excel (.xlsx/.xls) وPDF وصور (PNG/JPG) مع OCR، ويحفظ البيانات المستخرجة كقالب جاهز تلقائياً.

## التغييرات المطلوبة

### الملف: `src/pages/CostAnalysisPage.tsx`

**1. إضافة حالة (State) جديدة:**
- `isImportingTemplate`: لعرض مؤشر التحميل
- `importTemplateDialogOpen`: لفتح حوار تسمية القالب المستورد
- `pendingTemplateItems`: لتخزين البنود المستخرجة مؤقتاً قبل الحفظ

**2. إضافة دالة `handleTemplateFileImport`:**
- تقبل ملفات: `.xlsx`, `.xls`, `.pdf`, `.png`, `.jpg`, `.jpeg`
- **Excel**: يقرأ الملف مباشرة باستخدام ExcelJS ويستخرج البنود
- **PDF**: يستخدم `extractTextFromPDF` ثم يرسل النص المستخرج إلى edge function `analyze-costs` لتحويله لبنود منظمة
- **صور (PNG/JPG)**: يحول الصورة لـ base64 ويرسلها لـ edge function `ocr-extract` لاستخراج النص ثم يحلل النتيجة

**3. إضافة دالة `parseExtractedTextToItems`:**
- تحلل النص المستخرج (من PDF/OCR) وتحوله لمصفوفة `CostItem[]`
- تبحث عن أنماط: اسم بند + رقم إنتاجية + رقم إيجار
- تدعم النصوص العربية والإنجليزية

**4. إضافة دالة `saveImportedTemplate`:**
- تأخذ البنود المستخرجة + اسم القالب المدخل
- تنشئ `CostTemplate` جديد وتحفظه في `savedTemplates` و `localStorage`

**5. تعديل واجهة قسم "قوالب التحليل" (سطر 1160-1222):**
- إضافة زر "استيراد قالب" بجانب زر "حفظ كقالب جديد"
- الزر يفتح file picker يقبل `.xlsx,.xls,.pdf,.png,.jpg,.jpeg`
- عرض مؤشر تحميل أثناء المعالجة

**6. إضافة Dialog لتسمية القالب المستورد:**
- يظهر بعد نجاح الاستخراج
- يعرض عدد البنود المستخرجة
- يطلب اسم القالب من المستخدم
- أزرار: "حفظ كقالب" و "تحميل بدون حفظ"

### سير العمل (Workflow):
