

# تحسين التعامل مع الأسماء المكررة عند حفظ المشاريع

## المشكلة الحالية
مكون `SaveProjectDialog.tsx` يمنع الحفظ نهائياً عند وجود اسم مكرر (يعرض رسالة خطأ ويعطل زر الحفظ). المطلوب: السماح بالحفظ مع خيار **استبدال المشروع القديم وحذفه** أو **حفظ باسم جديد**.

## التغييرات المطلوبة

### الملف: `src/components/SaveProjectDialog.tsx`

1. **إزالة منع الحفظ عند التكرار**: بدلاً من تعطيل زر الحفظ، السماح بالضغط عليه حتى مع وجود اسم مكرر

2. **إضافة AlertDialog للتأكيد عند التكرار**: عند الضغط على "حفظ" واكتشاف اسم مكرر:
   - عرض حوار تأكيد يحتوي على:
     - **"استبدال وحذف القديم"** (زر أحمر): يحذف المشروع القديم بالكامل (من `saved_projects` + `project_data` + `project_items`) ثم يحفظ الجديد
     - **"حفظ باسم جديد"** (زر ثانوي): يضيف الطابع الزمني للاسم تلقائياً ويحفظ
     - **"إلغاء"**: يغلق الحوار

3. **تحديث دالة `saveProject`**: فحص التكرار عند الضغط على حفظ (وليس أثناء الكتابة فقط) وعرض حوار التأكيد

4. **إضافة دالة `handleOverwriteAndDelete`**: تحذف المشروع القديم من الجداول الثلاثة ثم تحفظ الجديد

5. **الاحتفاظ بمؤشر التحقق المباشر**: الأيقونات (علامة صح / تحذير) تبقى كمؤشر بصري أثناء الكتابة، لكن لا تمنع الحفظ

### الملف: `src/components/SaveProjectButton.tsx`

6. **تحديث `handleOverwrite`**: التأكد من حذف المشروع القديم بالكامل (items + data + saved) قبل إعادة الحفظ (الكود الحالي يفعل هذا بالفعل - تأكيد فقط)

## التفاصيل التقنية

### منطق الاستبدال والحذف

```text
1. حذف project_items المرتبطة بالمشروع القديم
2. حذف من project_data 
3. حذف من saved_projects
4. حفظ المشروع الجديد بنفس الاسم
```

### تغيير شرط canSave

```text
الحالي:  canSave = name.length > 0 && !nameExists && !isCheckingName
الجديد:  canSave = name.length > 0 && !isCheckingName
         (nameExists لم يعد يمنع الحفظ، بل يُظهر حوار التأكيد)
```

### حالات AlertDialog الجديد

```text
اسم مكرر → handleSave يفتح AlertDialog بدل رسالة خطأ
  ├─ "استبدال وحذف القديم" → handleOverwriteAndDelete()
  ├─ "حفظ باسم جديد"      → handleSaveWithNewName()  
  └─ "إلغاء"               → إغلاق الحوار
```
