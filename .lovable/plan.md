
# تحسين شاشة المستخلصات وربطها بالعقود والدفعات

## 1. تحسين شاشة إنشاء مستخلص جديد (`NewCertificatePage.tsx`)

### أ) عرض تفاصيل العقد المرتبط
عند اختيار عقد، يتم عرض بطاقة معلومات شاملة تتضمن:
- اسم العقد ورقمه ونوعه
- قيمة العقد وشروط الدفع ونسبة الاحتجاز
- مدة العقد (تاريخ البداية والنهاية)
- شروط العقد المهمة (Terms & Conditions) إن وجدت
- نسبة الإنجاز الحالية للعقد (`execution_percentage`)

### ب) ربط بالدفعات السابقة
- بعد اختيار العقد، جلب `contract_payments` المرتبطة بنفس العقد
- عرض بطاقة "سجل الدفعات" تتضمن: عدد الدفعات، المدفوع، المعلق، المتأخر
- ربط المستخلص تلقائيا بأقرب دفعة معلقة (milestone)

### ج) تحسين التصميم البصري
- استخدام `PageHeader` الموحد بدلا من العنوان المحلي
- تحسين بطاقات الملخص المالي بإضافة أيقونات وتدرجات
- إضافة شريط تقدم مقارن (الحالي vs قيمة العقد)
- تحسين جدول البنود بإضافة ترقيم صفحات (pagination) لتحسين الاداء عند وجود بنود كثيرة (عرض 25 بند مع "عرض المزيد")

- تحسين حوار العرض (View Dialog) بإضافة تفاصيل العقد المرتبط
- إضافة بطاقة "الدفعات" في حوار العرض

### ج) إضافة إجراءات سريعة
- زر "تصدير الكل PDF" لتصدير جميع المستخلصات المفلترة
- إضافة فلتر بالعقد (بالإضافة للمشروع والمقاول)

---

## 3. ربط المستخلصات بالعقود والدفعات

### البيانات المطلوبة (موجودة في قاعدة البيانات):
- `contracts` - العقود (contract_value, retention_percentage, terms_conditions, payment_terms)
- `contract_payments` - الدفعات (amount, status, due_date)
- `contract_milestones` - المراحل (milestone_name, payment_percentage, status)
- `progress_certificates` - يحتوي بالفعل على `contract_id`

### المنطق:
1. عند اختيار مشروع + مقاول، يتم جلب العقود المرتبطة (موجود)
2. عند اختيار عقد، يتم جلب الدفعات والمراحل
3. عرض شروط العقد (من `terms_conditions` و `payment_terms`)
4. حساب المتبقي من قيمة العقد بعد خصم المستخلصات السابقة
