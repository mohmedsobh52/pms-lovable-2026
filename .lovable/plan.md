
# خطة: إصلاح شاشة تحليل التكاليف + اقتراحات + دعم Excel متعدد الشيتات

## المشاكل المكتشفة

### 1. مشكلة الرسم البياني - عرض `[object Object]`
الرسم البياني (PieChart) يعرض `[object Object]` بدلاً من أسماء البنود. السبب: مكون `CustomTooltip` معرّف كدالة عادية بدون `forwardRef` مما يسبب تحذير React، وبيانات Legend تعرض كائنات بدلاً من نصوص.

### 2. تحذير Console: `Function components cannot be given refs`
المكون `CustomTooltip` في السطر 970 يحتاج تحويله لمكون عادي أو إضافة `forwardRef`.

### 3. بطء التحميل
الملف يحتوي 1511 سطر مع imports ثقيلة (dnd-kit, recharts, jspdf, exceljs) كلها تُحمَّل مباشرة.

## الحلول

### الملف: `src/pages/CostAnalysisPage.tsx`

**1. إصلاح `CustomTooltip`** (سطر 970)
- تحويل المكون ليكون خارج الـ render function وبدون ref

**2. إصلاح بيانات الرسم البياني** (سطر 781-809)
- التأكد من أن `chartData` يحتوي على `name` كنص وليس كائن
- إصلاح Legend formatter ليعرض النص بشكل صحيح

**3. إضافة قسم اقتراحات ذكية**
- إضافة بطاقة "اقتراحات وتوصيات" في الشريط الجانبي (العمود الثالث)
- اقتراحات ديناميكية:

| الشرط | الاقتراح |
|---|---|
| لا توجد بنود محللة بـ AI | استخدم تحليل AI السريع لتقدير الإنتاجية والإيجار |
| لا توجد قوالب محفوظة | احفظ إعداداتك كقالب لإعادة استخدامها |
| نسبة الهالك = 0 | أضف نسبة هالك لتقدير أدق للتكلفة |
| لا يوجد ربط مع البنود | استورد بنود من تحليل BOQ الرئيسي |
| يوجد فرق كبير بين AI والقيم اليدوية | راجع البنود ذات الانحراف الكبير |

**4. دعم رفع ملف Excel متعدد الشيتات (20+ شيت)**
- تعديل `handleFileImport` لعرض قائمة الشيتات المتاحة عند رفع ملف متعدد
- إضافة Dialog لاختيار شيت أو أكثر
- عرض معاينة سريعة لعدد الصفوف في كل شيت
- دعم استيراد من شيت واحد أو دمج عدة شيتات

**5. إضافة نصائح الصفحة (Page Tips)**

### الملف: `src/lib/page-tips.ts`
إضافة tips لمسار `/cost-analysis`:
- "أضف بنود التكاليف يدوياً أو استوردها من ملف Excel"
- "استخدم تحليل AI لتقدير الإنتاجية والإيجار اليومي"
- "احفظ إعداداتك كقالب لإعادة استخدامها في مشاريع أخرى"
- "صدّر التقرير بصيغة Excel أو PDF من القائمة الجانبية"
